Display: Inline, Block, and Inline-Block

Inline Elements

An inline element occupies only the space of the content. It does not take width and height values.

Example: anchor elements

Anchor elements are normally inline. They only take up the width of their content, not the whole width of the page. One inline element follows another on the same line.

Floating an inline element will change its display value to inline-block. As such, it will take margin, padding, height, and width. Block Elements

Block level elements take up the entire width of a page and always start a new line. It accepts margin and padding values. If no width is set, it will expand to fill its parent container. If no height is set, it will expand to fit its content.

Floating block elements.

When floated, a block element becomes inline-block. Unless width is specified, the element will shrink to the size of the content.

Inline-Block Elements

Inline-block elements take all the box-sizing properties of block elements, but sit on one line. They appear similar to elements with the float: left property. A few differences:

  • Surrounding content does not wrap around the elements; no clearfix required.
  • Inline-block elements do not respond to viewport size, but floated elements do.
  • Whitespace is added between the elements. It cannot be removed through css; the html tags must be written on the same line, or line breaks can be commented out. The following anchor elements have no margin, so this whitespace can be easily seen.

Clearing floats:

Choose from the following clear-fixes to make sure other elements do not wrap around floated elements:

  1. Place an element below the floated elements, with the property clear: left/right/both
  2. Place the floated elements in a container, and add an ::after pseudo-element to the container with a clear property. Any pseudo-element should also have a content property.

    container::after {
    content: "";
    display: block;
    clear: both;
    }

  3. Add overflow: hidden to the parent container. Besides clearing floats, this causes the parent to expand to the size of the floated elements.

Positioning

For more precise positioning than floats can accomplish, we can use the position property. This property can accept the following values:
  • Static
  • Fixed
  • Relative
  • Absolute

Static Positioning

The element renders normally on the page. This is the default position.

Fixed Positioning

A fixed-positioned element will stay in the same place even when the page is scrolled. top, right, bottom, and left are the properties used to position the element. Fixed-positioned elements are taken out of the page flow and do not leave a gap in the page where they would've been.

I'm positioned fixed.

Relative Positioning

With this property, setting offset values will move an element away from its default position. This sentence has position: relative; top: 5px; and a background color.An empty space will be preserved where the element would've normally been located on the page.

Absolute Positioning

Absolute-positioned elements are moved in relation to their closest relatively positioned parent element (verses fixed-positioned elements which are always positioned in relation to the viewport). If there is no relatively positioned parent, they are positioned within the body element.
